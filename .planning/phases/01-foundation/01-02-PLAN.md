```css
@import "tailwindcss";

@theme {
  /* Custom colors — generates bg-gamio-*, text-gamio-*, border-gamio-* utilities */
  --color-gamio-bg:           #0a0a0a;
  --color-gamio-bg-secondary: #111111;
  --color-gamio-orange:       #FF6B35;
  --color-gamio-orange-dark:  #CC4400;

  /* Font — references CSS variable injected by next/font on <html> element */
  --font-sans: var(--font-sora), 'Space Grotesk', system-ui, sans-serif;

  /* Width tokens */
  --width-container: 1440px;
  --width-inner:     1280px;
}

/* Raw CSS variables for use in arbitrary Tailwind values and GSAP color targets */
:root {
  --gamio-bg:           #0a0a0a;
  --gamio-bg-secondary: #111111;
  --gamio-orange:       #FF6B35;
  --gamio-orange-dark:  #CC4400;
  --gamio-purple-bg:    #1a0028;
  --gamio-purple-end:   #2d0045;
  --gamio-glass:        rgba(255, 255, 255, 0.05);
  --gamio-glass-border: rgba(255, 255, 255, 0.10);
  --gamio-text:         #FFFFFF;
  --gamio-text-muted:   rgba(255, 255, 255, 0.70);
  --gamio-text-subtle:  rgba(255, 255, 255, 0.60);
}

body {
  background-color: var(--gamio-bg);
  color: var(--gamio-text);
}
```

Key points:
- `@import "tailwindcss"` is the ONLY Tailwind directive (v4 syntax). Do NOT add `@tailwind base`, `@tailwind components`, or `@tailwind utilities` — those are v3 directives.
- The `@theme` block generates utility classes: `bg-gamio-orange`, `text-gamio-bg`, etc.
- The `:root` block exposes variables for arbitrary CSS values (`bg-[var(--gamio-glass)]`) and for GSAP animation targets in Phase 3.
- `--gamio-glass` and `--gamio-glass-border` are rgba values — they go in `:root` only, NOT in `@theme`, because Tailwind v4 does not reliably generate utilities from rgba color values.
- Do NOT create a `tailwind.config.js` file. The UI-SPEC Tailwind Config Contract (lines 136–161) is in v3 syntax — this is a documentation artifact. The token VALUES have been translated to v4 `@theme` syntax above.

```typescript
import type { Metadata } from 'next'
import { Sora } from 'next/font/google'
import './globals.css'

const sora = Sora({
  subsets: ['latin'],
  weight: ['400', '700'],
  variable: '--font-sora',
  display: 'swap',
})

export const metadata: Metadata = {
  title: 'Gamio',
  description: 'Game & Gain — B2B iGaming platform',
}

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en" className={sora.variable}>
      <body>{children}</body>
    </html>
  )
}
```

Key points:
- `weight: ['400', '700']` is mandatory — Sora is NOT a variable font. Omitting weights causes a build error.
- `variable: '--font-sora'` injects a CSS variable on the `<html>` element that `globals.css @theme --font-sans` references via `var(--font-sora)`.
- `className={sora.variable}` goes on `<html>`, NOT on `<body>`.
- `import './globals.css'` must be in layout.tsx (App Router convention for global styles).
- Do NOT add a `<link>` tag for Google Fonts — `next/font/google` self-hosts automatically.
- Do NOT add `'use client'` — this is a Server Component.
